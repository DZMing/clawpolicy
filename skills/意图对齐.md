# 意图对齐 Skill

这个Skill通过观察你的工作模式，学习你的偏好，帮助OpenClaw更好地为你服务。

## 核心功能

### 1. 偏好学习

**观察内容**：
- 任务类型偏好（T1/T2/T3/T4分布）
- 技术栈选择（React vs Vue, FastAPI vs Express）
- 代码风格偏好（命名规范、注释风格）
- 交互风格（直接 vs 详细、简洁 vs 完整）

**学习方式**：
- ✅ **隐式学习**：从你的操作中自动提取偏好
- ✅ **显式确认**：关键决策时主动询问
- ✅ **渐进式优化**：每次使用后微调理解

### 2. 模式识别

**自动识别**：
- 常用工作流（如：测试驱动开发、先写后测）
- 决策模式（如：优先性能 vs 优先可读性）
- 禁忌事项（如：从不使用某个库、拒绝某种模式）

**应用场景**：
- 新任务开始时，自动推荐符合你偏好的方案
- 多选决策时，优先选择历史倾向选项
- 检测到潜在禁忌时，主动提醒

### 3. 自适应优化

**优化维度**：
- **沟通风格**：学习你喜欢的汇报格式和详细程度
- **决策边界**：理解哪些决定需要你确认，哪些可以自主
- **自动化水平**：根据任务复杂度动态调整自主权

**优化时机**：
- 每次任务完成后，微调偏好模型
- 检测到行为模式变化时，主动询问是否更新
- 每周主动询问是否需要深度优化

```json

  "version": "1.0.0",
  "last_updated": "2026-02-28T18:00:00Z",
  "preferences": {
    "automation_level": "balanced",
    "communication_style": "direct",
    "tech_stack": {

    }
```

## 技术实现

### 数据收集

**来源**：
- Git commit 历史（技术栈选择、代码风格）
- 任务执行记录（决策模式、自动化边界）
- 用户反馈（显式确认、修改建议）

**隐私保护**：
- ✅ 只存储在本地
- ✅ 不上传到云端
- ✅ 可以随时清空

### 模式识别

**算法**：
- 简单频率统计（如：80% 使用 React）
- 序列模式挖掘（如：总是先测试后实现）
- 异常检测（如：突然使用新库）

**应用**：
- 下次遇到类似场景时，优先推荐历史偏好
- 检测到异常时，主动询问是否改变偏好

### 自适应优化

**策略**：
- **保守优化**：每次只微调一个参数
- **快速回滚**：如果用户不满意，立即恢复
- **定期审查**：每周询问是否需要深度优化

## 常见问题

### Q: 这个Skill会替我决策吗？

A: 不会。它只是学习你的偏好，在下一次需要决策时，优先推荐符合你历史的选项。最终决定权永远在你手中。

### Q: 我的数据会去哪里？

A: 所有数据都存储在本地 `~/.openclaw/extensions/intent-alignment/`，不会上传到任何地方。

### Q: 如何重置学习？

A: 删除配置文件夹即可：`rm -rf ~/.openclaw/extensions/intent-alignment/`

### Q: 如何查看当前学到的偏好？

A: 查看配置文件：`cat ~/.openclaw/extensions/intent-alignment/config.json`
